### Задание 1

`Задание 1. СУБД
Кейс
Крупная строительная компания, которая также занимается проектированием и девелопментом, решила создать правильную архитектуру для работы с данными. Ниже представлены задачи, которые необходимо решить для каждой предметной области.

Какие типы СУБД, на ваш взгляд, лучше всего подойдут для решения этих задач и почему?`

1 `Бюджетирование проектов с дальнейшим формированием финансовых аналитических отчётов и прогнозирования рисков. СУБД должна гарантировать целостность и чёткую структуру данных.`
2 `Под каждый девелоперский проект создаётся отдельный лендинг, и все данные по лидам стекаются в CRM к маркетологам и менеджерам по продажам. Какой тип СУБД лучше использовать для лендингов и для CRM? СУБД должны быть гибкими и быстрыми.`
3 `Отдел контроля качества решил создать базу по корпоративным нормам и правилам, обучающему материалу и так далее, сформированную согласно структуре компании. СУБД должна иметь простую и понятную структуру.`
4 `Департамент логистики нуждается в решении задач по быстрому формированию маршрутов доставки материалов по объектам и распределению курьеров по маршрутам с доставкой документов. СУБД должна уметь быстро работать со связями.`

### Ответ
1: `Лучше всего подойдёт реляционная СУБД (PostgreSQL, MySQL, Oracle).

Финансовые данные требуют строгой структуры, целостности и поддержки ACID-транзакций. Реляционная модель обеспечивает чёткую схему данных с таблицами, связями и ограничениями, что критически важно для бюджетирования — нельзя допустить, чтобы данные оказались в неконсистентном состоянии. PostgreSQL дополнительно хорош мощными аналитическими функциями (оконные функции, CTE), что полезно для формирования финансовых отчётов и прогнозирования рисков.`

2: `Для лендингов лучше использовать документоориентированную NoSQL СУБД (MongoDB, CouchDB) — структура данных лидов может отличаться от проекта к проекту, схема гибкая, важна скорость записи.

Для CRM лучше подойдёт реляционная СУБД (PostgreSQL, MySQL), так как данные о клиентах, сделках, воронках продаж имеют чёткую структуру и множество связей между сущностями.
`
3: `Лучше подойдёт документоориентированная СУБД (MongoDB). Учитывая, что структура формируется согласно иерархии компании (департаменты, отделы, должности), документы можно организовать в коллекции, вложенные структуры интуитивно понятны. Также можно рассмотреть иерархическую СУБД для прямого отображения структуры компании.`

4: `Лучше всего подойдёт графовая СУБД (Neo4j, ArangoDB). Задачи маршрутизации и распределения — это классические графовые задачи: объекты (склады, стройплощадки, офисы) — это узлы, маршруты между ними — рёбра. Графовые СУБД оптимизированы для обхода связей и поиска кратчайших путей, что идеально для логистики.`

### Задание 4 Кластеры

`Необходимо производить большое количество вычислений при работе с огромным количеством данных, под эту задачу выделено 1000 машин.

На основе какого критерия будете выбирать тип СУБД и какая модель распределённых вычислений здесь справится лучше всего и почему?`

### Ответ
В качестве СУБД для такого кластера подойдут:

Apache Cassandra — AP-система для высокой доступности.
Apache HBase — CP-система для строгой консистентности.
ClickHouse — для аналитических задач.

Оптимальная модель распределённых вычислений: MapReduce
